# 简单聊聊工作

在我刚毕业的时候，我是无法想象两年后的我是什么样子的，那个初入职场的我是青涩的，对未来充满迷茫的。而两年后的我，也就是现在的我，在这家公司有了很大的成长，这两年的工作，对我来说有很大的帮助。

### 初入职场

我很庆幸自己刚毕业就进入了Web酒店前端的团队，业务稳定且核心，能够让我慢慢适应工作的节奏，从学生思维转变为职场思维。

在这段时间里，我开始写线上H5站点的需求，Java+JSP的写法，非常老的技术栈，但也有种亲切感和熟悉感，大学第一次上的编程课语言就是Java，第一次写网页就是Jsp。

在这段时间里，我开始写微信小程序的需求，还记得大三和同学一起参加了微信小程序开发大赛，虽然落选但也熟悉了微信小程序的写法，写起来也算是得心应手。

在这段时间里，我也开始了解各类其他小程序，比如支付宝小程序、百度小程序、字节小程序、快应用。这是我之前所不涉及的，我开始尝试把微信小程序的代码转成各端小程序，并做些缝缝补补的改动。

在这段时间里，我开始写Online站点的需求，写的不多，这个站点还是React Component的写法，view.js和index.js分离的写法。这里不得不提，前端的框架变化真的太快，之前还在流行的写法很快就会被淘汰。

但是，工作不仅仅是写需求，还有各种会议要参加：需求澄清会、需求计划会、技术分享会...还有各种iq(用户反馈)要处理，还有和各方团队的沟通，还有每周三周四的封版和发布（熬夜的主要日子），还有很多很多...

这个时候，对我来说，需求发布上线就结束工作了。但是慢慢的，我开始关注数据，开始看监控，看埋点，看报错量，看PV和UV，看订单量...**开始知道我在做什么，我在为了什么而做事，这很重要。**

然后，我开始做重构。**重构是很大的挑战，但也很有意义。** 花了半年多，我的mentor带着我一起重构H5站点，全面地了解了整个酒店主流程的业务逻辑，从订前到订后，这也是我第一次这么深入地参与一个站点从搭建运行、到联调测试，到切流量上线、到数据监控、到性能优化的完整过程，目前为止，这个应用对我来说还是“白月光”一样的存在。

### 走向国际化

疫情即将结束的时候，公司暂缓的**国际化**项目又启动了，虽然我的组织架构没变，但人却在海外酒店前端的团队，这是一次全新的体验。

国际化是一个蛮难做的事：
- 一个是翻译问题，到目前为止，公共还在讨论和设计新的Shark方案，重复翻译和翻译不完全准确是两大问题；
- 一个是交互和展示的问题，不同国家不同地区的用户行为是不一样的，很难设计出完美的方案，代码里是各种特殊处理；
- 一个是服务器的问题，海外访问如果指向国内的集群就会很慢，现在的解决方案是海外区域新设机房和AWS上云并行，但我们还是很难模拟海外用户真实的访问情况；

另外就是这段时间，海外H5和Online的站点同步在做BFF技改和AB，这就导致一个组件的改动需要修改四遍，测试四遍。这很难，但难也要迎难而上嘛，这是一个阵痛期，稳定后就会好了。

So，BFF的站点也是前端在写，nodejs的写法。

### 创新项目

时间太晚了，细节下次有空补。

简单来说，这是和上面两个团队完全不一样的体验，是的，又一次全新的体验。
- 一个是因为负责的业务不同，不再是酒店主流程，而且各种地图的展示和交互；
- 一个是需求迭代节奏的不同，敏捷开发，快速交付和上线；
- 一个是我个人的成长，我开始做独立的项目，独立搭建站点并上线；我开始做校招生的导师，做Mentor.
